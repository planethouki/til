## Secret Lock Transaction

秘密の値を使ってモザイクをロックするためのトランザクション。このトランザクションには受信者のアドレスが含まれており、ロックが解除された場合、その受信者へモザイクが送信される。また、このトランザクションには有効期間のデータも含まれており、この期間内にロックが解除されない場合、モザイクは送信者へ返却される。

モザイクのロックは、秘密の値とそのハッシュを用いる。トランザクションには、秘密の値のハッシュが含まれており、このハッシュは誰でも見ることができる。一方、このハッシュの元となる秘密の値は含まれていないが、シークレットプルーフトランザクションにてその秘密の提出することができればロックが解除される仕組みになっている。

以下の値を使ってトランザクションを作成した。

|:-----:|:-----:|
| ロックモザイクID | `77A1969932D987D7` |
| ロックモザイク量 | `1000000` |
| ロック期間 | `960` |
| ハッシュ関数 | SHA3 256 |
| 秘密の値のハッシュ | ランダム生成 |
| 受け取り人 | ランダム生成 |

## Modify Multisig Transaction

アカウントをマルチシグ化するためのトランザクション。このトランザクション単体で送信するケースは少なく、アグリゲートトランザクションでラップして送信することが多い。

マルチシグアカウントの設定変更を行うのもこのトランザクションを用いる。変更には、署名アカウントの追加や削除、必要署名数の変更などがある。変更を行う際は、設定の絶対値ではなく、差分を指定する。

マルチシグアカウントの変更を行う場合は、アグリゲートトランザクションでラップして必要数の連署が必要となるが、アカウントを新たにマルチシグ化する場合においても、連署人の署名が必要となる。これには理由があり、ひとつのアカウントは複数のマルチシグアカウントの連署者になれるが、この数は制限されており、ある一定の数になると連署者として新たに指定することができなくなる。これを悪用すると、特定のアカウントをマルチシグの連署者にすることができなくするような攻撃が想定される。これを防ぐために、アカウントを新たにマルチシグ化する場合においても連署人の署名が必要となる。
